I learnt about description list (dl) and how to use it to create a section in the webpage that is similar to a glossary; I used the dl element to do so. Also, just because an element gives similar
appearances when used on HTML doesn't mean we use them interchangeably. An example is the <em> and <i>; both italicize the words, but the semantics are different. The browser interprets these two according to semantics. One cool thing I was thinking about as I code is that coding is leadership. We find ourselves emphasising on words so that your people who are browsing can see it as they read. There is a journey, and there are landmarks. Make sure you love others and desire a great experience for them. Let them not miss out on important things on a website or app.

Today I learnt about the block quotation and inline quotation as a way of quoting from books, websites, journals, etc. The cite attribute used in the "blockquote" and "quote" points to the resource being quoted. There is a "cite" element as well that does highlight a quoted text visibly on the webpage. In the other elements( by other, I mean blockquote and quote(q)), there is no evidence when it comes to presentation that the block or statement is quoted. In the cite element, the text is italicized
